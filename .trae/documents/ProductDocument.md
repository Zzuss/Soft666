# EBU6304 课程项目 - 产品需求文档

## 教学助理招聘系统（TA Recruitment System）

---

## 1. 项目概述

### 1.1 项目背景
BUPT国际学院每学期招聘教学助理（Teaching Assistant，简称TA）以支持学术模块和各种学校活动。目前申请流程依赖表单和Excel文件，效率较低。本系统旨在简化招聘工作流程。

### 1.2 项目目标
开发一个轻量级Web应用，用于BUPT国际学院TA招聘管理，支持TA申请人、课程负责人（MO）和管理员三类用户。

### 1.3 架构限制（必须遵守）
1. **开发语言/架构**：轻量级Java Servlet/JSP Web应用（禁止Spring Boot等框架）
2. **数据存储**：JSON文件格式（禁止数据库）

---

## 2. 用户角色

| 角色 | 描述 | 权限 |
|------|------|------|
| TA（申请人） | 申请成为教学助理的学生 | 创建个人资料、上传简历、查找职位、申请职位、查看申请状态 |
| MO（课程负责人） | 发布职位并选择申请人 | 发布职位、查看申请人、选择TA、查看已选TA的工作量 |
| 管理员 | 系统管理员 | 管理系统、查看所有TA工作量、导出数据 |

---

## 3. 功能需求

### 3.1 TA（申请人）功能

#### 3.1.1 注册与登录
- 用户注册：填写用户名、密码、姓名、邮箱
- 用户登录：使用用户名和密码登录
- 登出功能

#### 3.1.2 个人资料管理
- 创建/编辑个人资料（姓名、邮箱、专业、技能、可用时间）
- 上传简历（PDF格式，存储到服务器文件系统）
- 查看个人资料

#### 3.1.3 职位浏览
- 查看所有可用职位列表
- 按职位类型筛选（模块辅导、监考、其他）
- 查看职位详情（职位名称、描述、要求、截止日期）

#### 3.1.4 职位申请
- 对感兴趣职位提交申请
- 上传申请文件（如需要）
- 查看申请状态（待审核、通过、拒绝）

### 3.2 MO（课程负责人）功能

#### 3.2.1 职位发布
- 创建新职位（职位名称、类型、描述、要求、人数、截止日期）
- 编辑已发布的职位
- 关闭/删除职位

#### 3.2.2 申请人管理
- 查看职位申请人列表
- 查看申请人详细信息和简历
- 选择/拒绝申请人
- 发送通知给申请人

#### 3.2.3 TA管理
- 查看已选择的TA列表
- 查看TA的工作量统计

### 3.3 管理员功能

#### 3.3.1 用户管理
- 查看所有用户
- 重置用户密码
- 禁用/启用用户

#### 3.3.2 数据统计
- 查看所有TA的工作量统计
- 导出数据（CSV格式）

## 4. 数据模型

### 4.1 用户（User）
```json
{
  "userId": "用户ID",
  "username": "用户名",
  "password": "密码（加密存储）",
  "name": "姓名",
  "email": "邮箱",
  "role": "角色（TA/MO/ADMIN）",
  "skills": ["技能列表"],
  "availableTime": "可用时间",
  "createdAt": "创建时间"
}
```

### 4.2 简历（Resume）
```json
{
  "resumeId": "简历ID",
  "userId": "用户ID",
  "fileName": "文件名",
  "filePath": "文件路径",
  "uploadedAt": "上传时间"
}
```

### 4.3 职位（Job）
```json
{
  "jobId": "职位ID",
  "title": "职位名称",
  "type": "职位类型（MODULE/INVIGILATION/OTHER）",
  "description": "职位描述",
  "requirements": "职位要求",
  "positions": "招聘人数",
  "postedBy": "发布者ID",
  "deadline": "截止日期",
  "status": "职位状态（OPEN/CLOSED）",
  "createdAt": "创建时间"
}
```

### 4.4 申请（Application）
```json
{
  "applicationId": "申请ID",
  "jobId": "职位ID",
  "userId": "申请人ID",
  "status": "申请状态（PENDING/APPROVED/REJECTED）",
  "appliedAt": "申请时间",
  "reviewedBy": "审核人ID",
  "reviewedAt": "审核时间"
}
```

## 5. 核心功能优先级

### 第一轮迭代（Must Have）
1. 用户注册/登录
2. TA创建个人资料
3. MO发布职位
4. TA浏览职位
5. TA申请职位
6. MO查看和选择申请人

### 第二轮迭代（Should Have）
7. 简历上传/下载
8. 申请状态查看
9. 职位筛选和搜索
10. TA工作量统计

### 第三轮迭代（Nice to Have）
11. 管理员数据导出
12. 职位编辑和关闭
13. 通知功能

---

## 6. 非功能需求

### 6.1 性能要求
- 支持至少100个并发用户
- 页面响应时间 < 3秒
- 数据加载时间 < 2秒

### 6.2 可靠性
- 数据自动保存
- 异常处理和错误提示
- 文件上传大小限制（最大5MB）

### 6.3 安全性
- 密码加密存储（MD5）
- Session管理
- 权限控制
- 输入验证

### 6.4 易用性
- 直观的用户界面
- 清晰的导航
- 完善的错误提示
