# 数字化校园平台 - 产品需求文档

## 1. 项目概述

### 1.1 项目背景
本项目为EBU6304课程的小组作业，旨在开发一个数字化校园平台，为学校提供信息化的管理服务。

### 1.2 项目目标
构建一个轻量级的数字化校园管理平台，支持学生、教师、管理员三种角色的日常校园事务处理。

### 1.3 目标用户
- **学生**：使用校园服务、查询信息
- **教师**：管理课程、录入成绩
- **管理员**：系统管理、数据维护

## 2. 功能需求

### 2.1 用户管理模块
- **用户登录/登出**：支持三种角色登录
- **用户注册**：管理员可创建用户账号
- **个人信息管理**：查看和修改个人信息
- **密码修改**：支持修改登录密码

### 2.2 图书借阅管理模块
- **图书查询**：按书名、作者、ISBN搜索
- **图书借阅**：学生可借阅图书
- **图书归还**：归还已借图书
- **借阅记录**：查看个人借阅历史
- **图书管理**：管理员添加/删除/修改图书信息

### 2.3 课程管理模块
- **课程查询**：查看所有课程信息
- **选课功能**：学生选择/退选课程
- **课程管理**：教师创建/修改课程
- **学生名单**：教师查看选课学生

### 2.4 成绩管理模块
- **成绩录入**：教师录入学生成绩
- **成绩查询**：学生查看个人成绩
- **成绩统计**：查看平均分、排名等
- **成绩导出**：支持导出成绩单

### 2.5 校园卡管理模块
- **余额查询**：查询校园卡余额
- **充值功能**：为校园卡充值
- **消费记录**：查看消费历史
- **卡片管理**：挂失、解挂等操作

## 3. 非功能需求

### 3.1 性能需求
- 系统响应时间：普通操作 < 2秒
- 支持并发用户数：≥ 50人
- 数据读取速度：本地文件IO优化

### 3.2 可用性需求
- 界面简洁直观，易于操作
- 提供清晰的错误提示
- 支持基本的输入验证

### 3.3 安全需求
- 密码加密存储
- 用户权限控制
- 防止非法数据访问

### 3.4 可维护性需求
- 代码结构清晰
- 模块划分合理
- 便于功能扩展

## 4. 数据存储需求

### 4.1 数据格式
- 用户信息：JSON格式存储
- 图书信息：CSV格式存储
- 借阅记录：JSON格式存储
- 课程信息：JSON格式存储
- 成绩数据：CSV格式存储
- 校园卡数据：JSON格式存储

### 4.2 数据文件组织
```
data/
├── users.json          # 用户数据
├── books.csv           # 图书数据
├── borrow_records.json # 借阅记录
├── courses.json        # 课程数据
├── enrollments.json    # 选课记录
├── grades.csv          # 成绩数据
└── campus_cards.json   # 校园卡数据
```

## 5. 用户界面需求

### 5.1 界面风格
- 简洁清晰的布局
- 统一的视觉风格
- 友好的交互反馈

### 5.2 界面类型
- **控制台界面**：独立Java应用版本
- **Web界面**：Servlet/JSP版本

### 5.3 主要页面
- 登录页面
- 主页/仪表板
- 用户个人信息页
- 图书查询页
- 借阅管理页
- 课程列表页
- 成绩查询页
- 校园卡管理页
- 系统管理页（管理员）
